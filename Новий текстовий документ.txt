Контрольні питання:
1)В якому статусі можуть перебувати файли при роботі з Git? 
Untracked(Неконтрольований)
Unmodified(Немодифікований)
Modified(Модифікований)
Staged(Інсценований)
2.Для чого використовується команда add?
Для того щоб почати контролювати новий файл.
3. Для чого використовується команда status? 
Для того щоб подивитись повний список змін файлів, які очікують коммітов.
4. Для чого використовується команда commit?
Зберігає  зліпок файлів у внутрішній базі даних
5. Для чого використовується команда init? 
Для того, щоб створити новий репозиторій
6. Що необхідно писати в коментарях комітів? 
Потрібно писати змістовні коментарії,що було змінено та чому
7. Для чого використовується команда checkout? 
Щоб працювати над додатками
8. Для чого використовується команда reset? 
Використовується для скасування змін.
9. Для чого використовується команда log? 
Вона використовується, щоб показати досяжну записану історію проекту, починаючи з останнього відбитку коміту у зворотному порядку.
10.Для чого використовується команда status? 
Для того щоб подивитись повний список змін файлів, які очікують коммітов.
11.Як відновити кілька файлів з попередньо створеного коміта? 
git reset -- myfile
git checkout -- myfile
12.Як перейди до попереднього коміта по імені?
git log -S function_name
13.Як створити нову гілку? 
git branch
14.Як перейти на нову гілку?
git checkout нова гілка
15.Як перейти на основну гілку? 
git checkout master
16.Як виконати злиття гілок?  
git merge
